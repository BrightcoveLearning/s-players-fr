Le module Players permet d'intégrer facilement Brightcove Players à Google Analytics. Cela permet d'envoyer les événements des lecteur s et les données de dimensions personnalisées à un compte Google Analytics à des fins de création de rapports. Notez que lorsque l'intégration est configurée, seules les analyses des événements vidéo seront transmises à Google Analytics.

Remarque: Chaque système d'analyse collecte des données et calcule les mesures différemment. {{site.product_short}} les données d'analyse ne seront pas identiques à celles qui apparaissent dans Google Analytics.
Remarque: Le plug-in Google Analytics n'est pas pris en charge lors de l'utilisation des SDK du lecteur Brightcove.
Remarque: Le module Players prend en charge le modèle de données Universal Analytics de Google. La nouvelle mise à jour de Google Analytics 4 n'est pas prise en charge. Consultez la section Versions de Google Analytics ci-dessous pour plus d'informations.

Configuration de Google Analytics

Pour configurer l'intégration de Google Analytics, procédez comme suit:

  1. Ouvrez le module Players.
  2. Cliquez sur le nom du lecteur que vous souhaitez configurer pour ouvrir les propriétés du lecteur.
  3. Cliquez sur Third-Party Analytics > Google Analytics dans le menu de navigation de gauche.
  4. Cochez la case Activer Google Analytics.
  5. Cliquez sur Se connecter à un compte Google.
  6. Sélectionnez le compte Google avec lequel vous souhaitez vous connecter et saisissez vos informations d'identification si vous y êtes invité.
  7. Cliquez sur Autoriser pour permettre à Brightcove d'accéder à votre compte Google.
  8. Utilisez le menu déroulant Sélectionner un compte pour sélectionner le compte Google auquel envoyer les données de suivi.
  9. Sélectionnez les événements à suivre. Consultez la section Événements et dimensions personnalisées ci-dessous pour une description des événements.
  10. Cliquez sur Enregistrer. Cela configurera le lecteur avec le plug-in Google Analytics et créera toutes les dimensions personnalisées nécessaires dans votre compte Google Analytics.
    Remarque: Vous devez disposer de l'autorisation Modifier sur votre compte Google Analytics pour créer des dimensions.

Assurez-vous de publier les modifications sur le lecteur.

Remarque: Utilisez le code intégré avancé (in-page) lors de la publication de votre lecteur. Le code intégré standard (iframe) ne fonctionnera pas car le plugin s'exécutera à l'intérieur d'une iframe et ne pourra donc pas interagir avec le JavaScript nécessaire.

Événements et dimensions personnalisées

Les événements et les données de dimensions personnalisées peuvent être transmis à Google Analytics.

Les événements suivants peuvent être transmis.

Événements de base

  • Impression vidéo: envoyée lorsqu'une vidéo a été chargée dans un lecteur et qu'elle est prête à être lue
  • Demande de lecteur - Envoyée lorsque la lecteur est lancée soit lorsque l'utilisateur clique expressément sur le bouton de lecteur, soit automatiquement lorsque la plateforme déclenche la lecteur dans un scénario de lecteur automatique
  • Engagement vidéo - Un utilisateur a regardé une plage de secondes de la chronologie d'une vidéo. Cet événement est un battement de cœur pour le suivi de l'engagement vidéo et sera probablement envoyé de nombreuses fois pendant la lecteur, en fonction de l'interaction de l'utilisateur et de la durée de la vidéo.

Événements publicitaires

  • Ad Start - Envoyé lorsque le contrôle est transféré à un agent publicitaire par la plateforme de lecteur
  • Ad End - Envoyé lorsque le contrôle est transféré de l'agent publicitaire à la plateforme de lecteur

Événements avancés

  • Player Load: une session de lecteur a été lancée par un utilisateur final
  • Error - Envoyé en cas d'erreurs fatales qui perturbent l'expérience de lecteur

Dimensions personnalisées

Les dimensions personnalisées vous permettent de combiner des données Google Analytics avec des données non Analytics. Les dimensions personnalisées collectent des données que Google Analytics ne suit pas automatiquement. Les dimensions personnalisées suivantes sont disponibles.

Remarque: Pour plus d'informations sur les dimensions personnalisées, consultez l'aide de Google Analytics.
  • Video Seconds Viewed: nombre de secondes regardées (disponible uniquement si l'événement Video Engagement a été sélectionné)
  • Plage: plage de secondes visionnées (disponible uniquement si l'événement Video Engagement a été sélectionné)
  • Durée de la vidéo: durée de la vidéo en secondes (disponible uniquement si l'événement Video Engagement a été sélectionné)
  • ID du lecteur et nom du lecteur - ID et nom du lecteur
  • Nom du compte: ID de compte Brightcove
  • Session: identifiant de session ; la valeur de la session doit être constante pendant le cycle de vie d'une expérience et envoyée pour tous les événements de cette expérience
  • Version du lecteur: version du lecteur Brightcove

Versions de Google Analytics

Le module Players prend en charge le modèle de données Universal Analytics de Google. Actuellement, la nouvelle mise à jour de Google Analytics 4 n'est pas prise en charge. Brightcove recommande d'utiliser Universal Analytics lors de l'intégration au module Players et aux lecteurs Brightcove.

Vous trouverez ci-dessous les étapes de haut niveau pour créer une propriété Universal Analytics dans Google Analytics.

  • Créer une propriété
  • Afficher les options avancées
  • Basculer Créer une propriété Universal Analytics sur (l'interrupteur à bascule se trouve sur la droite)
  • Entrez l'URL d'un site Web
  • Assurez-vous que l'option Créer uniquement une propriété Universal Analytics est sélectionnée.
  • Remplissez le reste du formulaire ; vous recevrez un nouvel identifiant de propriété commençant par UA-

Notez que Google offre également la possibilité de créer une sélection hybride avec une propriété Google Analytics 4 et une propriété Universal Analytics. Brightcove n'a pas entièrement testé ce scénario.
