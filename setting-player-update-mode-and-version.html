L'un des principaux avantages du lecteur Brightcove est sa capacité à être personnalisé. La personnalisation permet d'utiliser des éléments et des couleurs d'interface utilisateur spécifiques à la marque, des éléments interactifs, l'intégration avec n'importe quel système de publicité ou d'analyse tiers, et bien plus encore. Mais la personnalisation peut également rendre le lecteur différent de la version testée par Brightcove dans le cadre du processus régulier d'assurance qualité et de publication. Pour les clients qui ont personnalisé leurs lecteurs et qui souhaitent s'assurer que les nouvelles versions ne perturberont pas le comportement du lecteur ou du site, Brightcove offre aux clients la possibilité de se désabonner des mises à jour automatiques régulières des versions.

Dans le cadre des paramètres du lecteur, le mode de mise à jour et la version du lecteur peuvent être configurés. Le mode de mise à jour contrôle la façon dont les mises à jour des lecteur s La version du lecteur est utilisée pour définir la version du lecteur. La possibilité de contrôler le mode de mise à jour et la version du lecteur offre les avantages suivants:

  • Les éditeurs peuvent essayer les versions préliminaires du lecteur sans avoir à utiliser l'API
  • Les éditeurs peuvent contrôler quand les mises à jour du lecteur sont effectuées

Processus de mise à jour

Brightcove s'engage à fournir les dernières fonctionnalités et corrections de bogues aux clients le plus rapidement possible. Dans le cadre de la discipline de livraison Agile, Brightcove s'efforce de diviser les améliorations des fonctionnalités et les corrections de bogues en mises à jour incrémentielles qui peuvent être publiées toutes les deux semaines. Le processus de publication comprend des tests unitaires, des tests d'intégration et de régression automatisés, ainsi que des tests manuels sur un échantillon représentatif de sites clients, où la page est modifiée localement dans l'environnement de test. Une fois qu'une nouvelle version a passé tous les tests, Brightcove la publie en tant que pré-version. Cela permet aux clients de choisir cette version explicitement en utilisant l'API de gestion du lecteur ou en utilisant le menu déroulant du sélecteur de version du lecteur dans le module Lecteur. À ce stade, aucun lecteur n'est mis à jour sauf ceux qui ont explicitement choisi la nouvelle version. Si vous utilisez le mode de mise à jour automatique par défaut et que vous voulez vous assurer que votre site fonctionnera correctement lors de la prochaine mise à jour globale, vous pouvez créer un lecteur de test spécial avec la nouvelle version et l'essayer sur une version test ou intermédiaire de votre site pendant la période de pré-lancement.

Mises à jour de versions

Les mises à jour de version majeures sont indiquées par une modification du premier chiffre dans l'identifiant de version, par exemple la version majeure de 6.42.5 est 6. Une modification apportée à la version majeure indique un changement important dans le fonctionnement d'une API, qui peut interrompre une intégration existante. Pour cette raison, et afin d'éviter tout problème avec votre site, aucune mise à jour automatique ne sera effectuée pour les mises à jour majeures des versions. Cela signifie que votre lecteur sera effectivement verrouillé lors de la sortie finale d'une version majeure, même si vous avez sélectionné le mode de mise à jour automatique.

Version gérée

Brightcove accorde généralement environ une semaine aux clients pour essayer une version préliminaire avant le lancement d'une version gérée. La version gérée est le processus qui consiste à exécuter un test A/B contrôlé sur les sites Web de production. Le système de publication géré permet de charger la nouvelle version du lecteur sur un faible pourcentage de pages chargées pendant une courte période, généralement quelques heures. Pendant ce temps, Brightcove collecte des données afin de comparer les performances des anciennes et des nouvelles versions. La distribution du nombre d'erreurs, du nombre de minutes visionnées, du nombre de lancements d'annonces et de plusieurs autres indicateurs sont tous examinés. Brightcove examine les comptes de distribution et les résumés des données, mais aussi les valeurs aberrantes, c'est-à-dire les sites qui présentent des différences supérieures à la moyenne dans les mesures suivies. Après un examen attentif de toutes les données, la mise à jour est validée ou annulée. S'il est décidé de l'annuler, de petites mises à jour peuvent être effectuées pour corriger un problème identifié, puis la version gérée est réexécutée.

Configuration du mode de mise à jour du

Pour modifier le mode de mise à jour du lecteur, procédez comme suit.

  1. Ouvrez le module Players.
  2. Cliquez sur un lecteur pour ouvrir ses propriétés.
  3. Cliquez sur Informations sur le lecteur dans le menu de navigation de gauche.
  4. Sélectionnez un mode de mise à jour. Les modes automatique et manuel sont disponibles.

    Mises à jour automatiques des

    Le mode automatique est le mode par défaut. Automatique provoque la mise à jour du lecteur chaque fois que Brightcove promeut une version du lecteur au statut par défaut actuel-fr/. Lorsque Automatique est sélectionné, les options Version du lecteur sont Toujours la dernière version et Toujours la dernière version préliminaire .

    Remarque: Pour plus d'informations sur la dernière version préliminaire du lecteur, consultez le document de préversion de Brightcove Player.

    Mises à jour manuelles

    En mode manuel, les lecteur s ne seront pas automatiquement mis à jour. C'est ce que l'on appelle également le verrouillage-fr/ du lecteur. Lorsque cette option est sélectionnée pour un lecteur , celui-ci ne fait pas partie du processus de publication géré et reste dans la version choisie. Vous devez bien réfléchir avant de choisir ce mode, car cela signifie que vous n'obtiendrez plus de corrections de bogues automatiques ni d'améliorations de fonctionnalités. À un moment donné, les anciennes versions du lecteur ne seront plus prises en charge. Vous devez utiliser le mode manuel uniquement comme solution temporaire pendant que vous résolvez les conflits entre la version existante du lecteur et la dernière version. Lorsque le mode Manuel est sélectionné, une version spécifique du lecteur peut être sélectionnée à l'aide de la liste déroulante Version du lecteur.

    Raisons de choisir Manual updates

    • Configuration avancée de la publicité - Si vous utilisez les plug-ins IMA ou FreeWheel et que vous avez rendu impossible toute configuration ou extension du comportement publicitaire via l'interface utilisateur de Studio, y compris la définition de paramètres de configuration personnalisés ou l'interaction avec le plugin à l'aide de JavaScript dans un plugin personnalisé ou logique de page, cela peut être une raison pour verrouiller votre lecteur.
    • Plug-ins tiers - Si vous utilisez un module d'analyse ou un autre plug-in fourni par un tiers, sachez que ces plug-ins ne sont pas gérés par Brightcove et qu'ils peuvent être sensibles à des modifications mineures du comportement des événements JavaScript du lecteur. Brightcove recommande aux utilisateurs de ces plug-ins de verrouiller leurs lecteurs.
    • Changements CSS importants - Des exemples de modifications CSS importantes incluent: la modification de la mise en page, du dimensionnement, du positionnement des éléments ou des modifications qui ne supporteraient pas les modifications de la mise en page, du dimensionnement ou du positionnement après une mise à jour. Exemples de modifications CSS sûres: modification du texte, des couleurs d'arrière-plan ou modification des polices d'une manière qui n'affecte pas la mise en page.
    Remarque: Lorsqu'un lecteur est verrouillé sur une version, les plugins associés à ce lecteur ne seront PAS automatiquement mis à jour, et également verrouillés sur la version actuelle pour chaque plugin.
  5. Cliquez sur Enregistrer.

Veillez à publier toutes les modifications apportées au lecteur.

Définition de la version du lecteur

La version du lecteur est utilisée pour définir la version du lecteur. Si le mode de mise à jour est Manuel et qu'une nouvelle version du lecteur est disponible, un message s'affiche.

Pour modifier la version du lecteur, procédez comme suit.

  1. Ouvrez le module Players.
  2. Cliquez sur un lecteur pour ouvrir ses propriétés.
  3. Cliquez sur Informations sur le lecteur dans le menu de navigation de gauche.
  4. Si le mode de mise à jour est défini sur Automatique, la version du lecteur peut être:
    • Toujours la dernière version - Cette option entraînera la mise à jour automatique de la version du lecteur vers la dernière version dès que de nouvelles versions seront disponibles
    • Toujours la dernière version préliminaire - Cette option entraînera la mise à jour automatique de la version du lecteur vers la dernière version préliminaire (ou la version finale si aucune version préliminaire n'est disponible) au fur et à mesure que de nouvelles versions préliminaires sont disponibles.
  5. Si le mode de mise à jour est défini sur Manuel, sélectionnez une version de lecteur spécifique.
  6. Cliquez sur Enregistrer.

Veillez à publier toutes les modifications apportées au lecteur.

Assistance

Le support technique de Brightcove fournira une assistance et des conseils à tout client disposant d'un contrat de support en cours de validité utilisant n'importe quelle version du lecteur Brightcove. Cependant, les corrections de bogues ne seront appliquées qu'à la dernière révision d'une version majeure. Après la publication d'une version majeure, certaines corrections de bogues seront appliquées à la dernière révision de la version majeure précédente pendant 6 mois maximum.

Brightcove espère que ces options vous fourniront les outils dont vous avez besoin pour garantir la meilleure expérience de lecteur possible sur votre site tout en fournissant le maximum d'options de personnalisation.
